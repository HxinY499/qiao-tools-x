# 2026-06-29

## SSE/JSONL 解析器优化

修复了两个问题：

1. **trailingIncomplete 标记** — `parseSse` 末尾 flush 前检测 `dataLines.length > 0`，若为 true 则在 `ParseResult` 中设置 `trailingIncomplete: true`。UI 端在 stats 区域显示黄色警告提示（中英 i18n 已添加）。

2. **isSignal 改为正则** — 移除硬编码 `KNOWN_SIGNALS` Set，改为正则 `/^\[[A-Z][A-Z_]*\]$/`，匹配所有大写字母+下划线的方括号信号（如 `[DONE]`、`[FINISHED]`、`[STREAM_END]`），同时不误判 JSON 数组或小写文本。

测试：新增 13 个测试用例，全部 36 个测试通过 + tsc 零错误。

## SSE/JSONL 解析器 — 第二轮优化

完成剩余 6 个优化点：

1. **looksLikeLjson 不再排除非 `{`/`[` 开头的行** — 移除快速排除逻辑，所有行都尝试 JSON.parse，支持原始值 JSONL（`"hello"`、`123`、`true`）。

2. **parseSse 不再 trim 行** — 严格遵循 SSE 规范，字段名必须从 column 0 开始。行首有空白的行不会被错误识别为字段行。处理 `\r\n` 行尾兼容。

3. **游标解析替代 split** — `parseSse` 和 `parseLjson` 均改用 `indexOf('\n', pos)` 游标方式逐行扫描，避免大文本一次性 split 产生的内存压力。

4. **全局粘贴去重** — `useBlockViewer` 中添加 `lastPasteRef`，相同内容重复粘贴不再触发重新解析和 toast。

5. **mergedJson 缓存优化** — 用 ref 缓存 blocks 引用，blocks 不变时直接返回旧值，避免重复 stringify。

6. **AUTO_COLLAPSE_THRESHOLD 配置化** — 移入 `settings-store.ts`（`autoCollapseThreshold`，默认 20，持久化），用 ref 传递给 handleParse callback。

7. **手动格式覆盖** — `parseStream` 新增 `forceFormat` 参数；UI 中加入 Auto/SSE/JSONL 三按钮切换，i18n 已补充。

测试：新增 8 个测试，全部 44 个通过 + tsc 零错误。
